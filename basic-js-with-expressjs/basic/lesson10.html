Lesson 10 : Object（オブジェクト）で表現の幅を広げる

Lesson 10 : Object（オブジェクト）で表現の幅を広げる

いよいよJavaScriptの基本の最終段階とも言ってよいObjectを学ぶ時が来ました。（おおげさ？）

前回、配列によってまとまった情報を表現することが出来ました。

が、少しスッキリしない感覚を覚えませんでしたか？（私は配列さえ居てくれたらそれでいいんだ！と言うのはぐっとこらえてくださいね。）

前回、以下のように生徒名と出席番号を無理矢理に表現しました。はい、無理矢理にです。

var students_name_and_number = ['Masahiro Nakai', 1, 'Goro Inagaki', 2, 'Takuya Kimura', 3, 'Shingo Katori', 4, 'Tsuyoshi Kusanagi', 5];

変数名から、「生徒の名前と番号を表してるんだなぁ」とか、[]内の記述から、「名前と番号を順番に書いて表現しているのか」などがわかるかと思います。

では、生年月日を追加したい時、あなたはどうしますか？性別や趣味、得意科目などを追加したい時は？

studentsname_number_birthday_gender...の様に変数名を書きます？['生徒名', 出席番号, '性別',,,]としますか？なかなか煩わしいですよね？

そこで便利なものが、Object(オブジェクト)です。まずは以下を見てみましょう。

var a_student = {
  name: 'Masahiro Nakai',
  number: 1,
  gender: 'male'
};

Objectは、{property: value}で表現します。propertyは、任意の名前。valueは、propertyに該当する値を意味します。

valueには、配列や関数も定義することができます。今は頭の片隅においておいてください。

ここでなんとなく、配列にオブジェクトも入れることが出来るのかな？と思えたら、良い傾向です。

とりあえず、まずはオブジェクトにアクセスする方法を体験してみましょう。

console.log(a_student.name); // 変数名.プロパティー名でアクセス可能です
console.log(a_student['name']); // 変数名['property名の文字列']でもアクセス可能です。なんだか配列みたいですね。

練習問題1
