Lesson 06 : レンダリングエンジンpug(jade)

Lesson 06 : レンダリングエンジンpug(jade)

前回までの内容だけでwebアプリケーションを作れないわけではないですが、GETリクエストで処理を変え、webページの内容も変化させて出力させたいことが多分に出てきます。

Expressにはいくつかのレンダリング（テンプレート）エンジンが使用できるように設定ができます。ここでは、デフォルトで設定されているpug(旧名：jade)の使い方について説明し、体験していきます。

ちなみに、jadeという名前がpugに変わった理由は商標登録問題のせいだそうです。jade時と使い方はさほど変わりません。

レンダリングエンジンがどのような効果をもたらしてくれるのでしょうか。

まずは使ってみてましょう。

pugの基本的な使い方と設定

$ npm install pug --save

index.jsにて、app.get等を記述する前に以下を設定します。

app.set('views', './views');
app.set('view engine', 'pug');

viewsディレクトリを作り、そこにindex.pugファイルを作り以下を記述します。

html
  head
    title= mytitle
  body
    h1= message

index.jsで、以下のように書き換えてみましょう。

app.get('/', function (req, res) {
  res.render('index', { mytitle: 'Hey', message: 'Hello there!'});
});

やっていることとしては、/（http://localhost:3000）にアクセスすると、index(.pug)ファイルに{ mytitle: 'Hey', message: 'Hello there!'}オブジェクトをパラメータとして渡してレンダリングし、それを返すということをしています。(ブラウザ上でソースコードを見るとhtmlに変換されていることが確認できます)

pugの文法ですが、Tabでインデントをつけることでhtmlタグのツリーを表現し、tag名を先頭に書きます。タグ内のテキストなどを変数で与えたい場合は、h1= [変数名(render()で渡したオブジェクトプロパティ名)]とします。

idやclassなどの設定方法

<div id="hoge">を表現する場合

div#hoge

<div class="foo">を表現する場合

div.foo

<div id="hoge" class="foo bar">を表現する場合

div#hoge.foo.bar

idやclass以外の<div style="border: solid 1px;">などを表現する場合

div(style="border: solid 1px;")

classなどを変数で表現する場合（たとえば、変数hensuuで値がhensuuvalueの場合）

div(class=hensuu)

とすると、これは以下と同じになります。

div.hensuuvalue

練習問題

  1. /, /about, /userそれぞれのhtmlページをpugを用いてレンダリングさせてみましょう
  2. /user/:type/:idの設定をし、ページを訪れた際にtypeとidが出力されるように設定しましょう
  3. /に訪れると、ページ上の数字が増えるようにしましょう
