Lesson 02 : NPM

Lesson 02 : NPM

NPMとは

npmとは、node package managerの略で、nodejsライブラリー（モジュール、パッケージ）を扱っていくためのマネージャーです。

webアプリケーションを構築していく上で、全てを０から１人でコーディングすることは時間もお金も体力も必要です。

webアプリケーションに限らず、ソフトウェアを構築する場合、ライブラリーとして共通化できる部分はたくさんあり使い回すことが出来ます。

そこでnode.jsに用意されたものがnpmとなり、npmを用いて世界中の人が構築してくれた便利なライブラリーをインストール、管理し使用してwebアプリケーションを構築していきます。

npmは、node.jsに同梱してインストールされますので、特別なインストール手順は必要ありません。

更に詳しい内容は公式ページを参照。

npm install

基本的に、npm install <package name>と打つだけで好きなパッケージをコマンドを入力したディレクトリにインストールすることができます。

たとえば、expressパッケージをインストールしたい場合、npmやgithubで公開されているページへ行き、Installationで紹介されているコマンドを入力すればインストール可能です。( npm install express )

node_modules

npm install <package name>をすると、node_modulesディレクトリが(もし、ない場合は)生成され、そこにインストールされたパッケージが入ります。

npm uninstall <package name>やnode_modules内の特定のパッケージを削除することでアンインストールは完了です。（要するに、installといってもコンピューター内に直接組み込まれるわけではなく、コマンド実行時のディレクトリにパッケージが置かれるだけになります。ただし、npm install -g express のようにグローバルオプションをつけてインストールすると、ローカルユーザーのnode_modulesディレクトリにインストールされます。詳細はこちら）

npm init と package.json

Node.jsアプリケーションを開発する際は、まず、npm initと打つことになります。そうすると、package.jsonという設定ファイルが生成されます。

ここには、ユーザー情報からライセンス情報、使っているパッケージを記述することになります。（npm init時に基本情報を求められますが、後で書き換えることもできるので必ず入力する必要はありません）

パッケージインストールの際、npm install express --saveのようにsaveオプションを指定することで、package.jsonに自動で追記してくれます。パッケージ情報の記述は必ず必要となるので、普段からこの癖をつけるようにしましょう。

npm install

たとえば、誰かが作成したアプリケーションをgithubなどからダウンロードしてきたとしましょう。そしてそれを動作させたい場合、npm installコマンドを打つことで、package.jsonに書かれた必要モジュールをインストールしてくれます。
