Lesson 08 : POST リクエスト

Lesson 08 : POST リクエスト

これまでの内容で、Webサイトは作れるようになりました。

Webアプリケーションとしてらしさをだすには、何が足りないでしょう。

普段webサイトへアクセスする場合は、ブラウザのアドレスバーへURLを入力し訪問します。これは、HTTP GETリクエストを使うことで実現できています。ユーザーである我々が、コンテンツを得る動作になります。では、その逆は出来ないのでしょうか、そしてどんなものが考えられるでしょうか。

一般的なものでいうと、お問い合わせフォームが思いつきますね。ユーザーがコンテンツを送ることができるのが、POSTリクエストになります。まずはコードを用意して動作を確認してみましょう。

body-parserをインストール(公式参照)

$ npm install --save body-parser

index.jsを以下のように変更。

var express = require('express');
var app = express();

var bodyParser = require('body-parser');
app.use(bodyParser.json());

app.use('/assets', express.static(__dirname + '/public'));

app.set('views', './views');
app.set('view engine', 'pug');

app.get('/', function(req, res) {
  res.render('index');
});

app.post('/', function(req, res) {
  var userName = req.body.userInput;
  res.send('Hello!' + userName + '!');
});

app.listen(3000, function() {
  console.log('Starting your application server at http://localhost:3000');
});

views/index.pugを以下のように設定

doctype html
html
  head
    title index sample
    script(src="assets/main.js")
  body
    p Hello visitor!
    p Could you tell me your name?
    input#userInput(type="text" placeholder="your name")
    Button#userSend Tell them
    div#responseServer

public/main.jsを標準Web APIインターフェースを使って書いてみましょう。

DOMを取得・操作するDocument

データ通信に使われるXMLHttpRequest

window.onload = function() {
  document.getElementById('userSend').addEventListener('click', function () {
    var userInput = document.getElementById('userInput').value;
    var req = new XMLHttpRequest();
    req.open('POST', '/', true);
    req.setRequestHeader('Content-Type', 'application/json');
    req.send(JSON.stringify({ userInput: userInput }));
    req.onreadystatechange = function () {
      if (req.readyState === 4 && req.status === 200) {
        document.getElementById('responseServer').textContent = req.responseText;
      }
    };
  });
};

準備ができたら、node index.jsで起動し、動作を確認しましょう。

これまでは、app.get()のみでしたが、app.post()を設置しました。HTTP GETではなく、HTTP POSTリクエストを受け付けるための設定になります。POSTで送信されてきた文字列とHello!を結合して返しています。

クライアント側（ブラウザ側）では、その文字列を送るようになっており、返ってきた文字列を<div id="responseServer"></div>に埋め込んでいます。

HTTP POSTによって、ユーザーからサーバー側へ任意の情報を送信し、そのレスポンスを受取る事が可能になりました。(気がついた方もいらっしゃるかもしれませんが、HTTP GETでも出来ないことはないです。ただし、GETではリクエストしたURLに情報が付加されるため、重要な情報を送る場合、セキュリティという観点では使うべきではありません。)

Content-Typeとは

POSTリクエストでデータを送る際、それがどのようなデータなのかを示す必要があります。

そこで重要になってくるのが、Content-Typeです。

フロント側では、application/json、ようするにJSONをデータとして送ろうとしています。

サーバー側では、POSTで送られてくるものをパースして上げる必要があり、デフォルトではその機能が無いのでbody-parserで別途インストールし、設定して上げる必要があります。（app.use(bodyParser.json());がJSONデータ用の設定）
