Lesson 05 : GET リクエスト

Lesson 05 : GET リクエスト

基本

前回、'/'へアクセスがあると、文字列を返してあげるだけでした。

それではつまらないですね。他へのルーティングも設定したいですし、文字列以外（htmlやjson、その他のファイル）を返したいです。

/ebisujsというルーティングを用意し、htmlを返してあげましょう。 index.jsと同じ階層に、ebisujs.htmlを作成し設置(教材サンプルからコピー : sample/get-request-sample/ebisujs.html)

以下を追加

// '/'の次に以下を追記
app.get('/ebisujs', function(req, res) {
  res.sendFile(__dirname + '/ebisujs.html');
});

node index.jsで起動していたプロセスをCtrl＋Cで止め、再度node index.js

http://localhost:3000/ebisujsへ行き、ebisujs.htmlの内容が表示されていれば完了。

このように、HTTP GETリクエストがあった場合のさまざまな設定を記述することが出来ます。

練習問題

公式ドキュメントを参考にしましょう。

  1. '/somedata'にアクセスするとjsonを返すようにしよう
  2. 自分で好きなようにルーティングを設定してみよう

様々なURLパラメータ

もしユーザーがいるようなアプリケーションで、ユーザーのタイプごとに応答内容を変えたい場合、URLのパラメータから処理を変える必要があります。

例えば、

/user/normal/sam, /user/normal/tom, /user/normal/bobなど、normalユーザーであったり、/user/admin/sara, /user/admin/eva, /user/admin/yuiなど、adminユーザーと分けたい場合が合ったとします。

すると以下のように設定することが可能です、

// :を先頭につけることで、req.paramsのプロパティに登録され、参照することが出来ます
app.get('/user/:type/:id', function(req, res) {
  if (req.params.type === 'normal') {
    // process for normal user
  } else if (req.params.type === 'admin') {
    // process for admin user
  }
});

皆さんがよく見るクエリーもしっかり処理することが出来ます。

先ほどのユーザータイプとIDをクエリーを使って表現するとこのような感じでしょうか、

/user?type=admin&id=tom

その場合の処理は以下のように書くことが出来ます。

app.get('/user', function(req, res) {
  if (req.query.type === 'normal') {
    // process for normal user
  } else if (req.query.type === 'admin') {
    // process for admin user
  }
});

練習問題

  1. req.paramsを使用した/user/:type/:idを設置して/user/normal/tomでアクセスした際のparamsの値を確認しましょう。
  2. req.queryを使用した/userを設置して`/user?type=admin&id=tomでアクセスした際のqueryの値を確認しましょう。
  3. reqパラメータからip, urlやpath, アクセスした人がhttpからアクセスしたのか、httpsからアクセスしたのかを調べてみましょう。
