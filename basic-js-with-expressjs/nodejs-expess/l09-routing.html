Lesson 09 : ルーティング

Lesson 09 : ルーティング

ミドルウェア設定

ルーティングに対してミドルウェアを設定することが出来ます。

例えば、/messageへのGETやPOSTへのルーティングを行う前に何かしらの処理がしたい場合、以下のように書けます。

app.use('/message', function (req, res, next) {
  console.log('Accessing the message section, then do something...');
  next(); // pass control to the next handler
});

/messageのパスに対して、上記の処理をマウントして実行することになります。

見てわかるようにfunction(req, res)の３番めにnextが設定されています。function(req, res, next)とすることで、next()は次のルーティング、例えばGETリクエストの場合はapp.get('/message')、へルーティングすることができます。

ちなみに、全てのルーティング(/ルート以降)に対するミドルウェアを設定したい場合は、/とするか、パスを省略すれば大丈夫です。

expressjsのサンプルを見ると、app.use([middleware]);と書かれているものを見ることが出来ます。

ルーティング用ファイル分割

アプリケーションの規模が大きくなってくると、全ての処理を1ファイルに書くと見にくいものになり、ファイルを分けたくなります。

その場合、ルーティング処理の記述は以下のように書くことが出来ます。

var user = require('path/to/user-routing-file');
app.use('/user', user);

path/to/user-routing-file（たとえば、./user.js）には、以下のように書けます。

var express = require('express');
var router = express.Router();

// define the home page route (!! this mean is like a 'localhost/user' as the home route !!)
router.get('/', function(req, res) {
  res.send('user top page (path is /user)');
});
// define the about route
router.get('/about', function(req, res) {
  res.send('About user (path is /user/about)');
});

module.exports = router;

本ページのミドルウェア設定で紹介した方法を使っていることがお分かりいただけたでしょうか。

index.js(メインで実行するファイル)では、app.use()で設定をし、その先でルーティングを書く事になります。

注意していただきたいのは、ルーティング設定のファイルでトップディレクトリをrouter.get('/user')のように書き始めないことです。設定ファイル(ここでは、user.js)では、router.get('/')のように/をトップディレクトリ（ルートディレクトリ）として書き、マウントする元（ここでは、app.use('/user', user)）から見たルーティングとして書きます。
