Introduction

Introduction

JavaScript / Node.js(Express) 教材

この教材では、JavaScript(js)の基礎をはじめ、Node.jsとNode.js製WebアプリケーションフレームワークのExpressの基礎を紹介します。

（JavaScriptの基礎を固めた上でNodejs/Expressjsへ進むことをお勧めします。）

それぞれのLessonでは、練習問題が用意されていることもあるので、是非自分で考えて取り組んでみてください。

JavaScript基礎について

JavaScriptを手軽に書いて実行するには、jsbinなどのブラウザ上でjsやhtmlを実行できるサービスを使う方法を提案します。

jsbinの場合、JavaScriptタブとConsoleタブのみを使えばよいです。(Outputタブは、HTMLタブで書いた内容が出力されるので注意が必要です)

Consoleタブを開くと、Console画面が表示されます。Console画面の右上に「Run」と「Clear」ボタンがあるので、JavaScript画面に記述したjsを実行するときは「Run」を、Console画面の出力をクリアする場合は「Clear」を押して下さい。

または、お手持ちのPCにNode.jsをインストールし、ターミナル（コマンドプロンプト）上で実行したりもできます。

Node.jsについて

Node.jsを扱う上で、ターミナル（コマンドプロンプト）の操作が少なからず必要になってきます。

使い方がわからない方は、以下の参考サイトをご覧の上でお進め下さい。特に、ディレクトリ（フォルダー）の位置（場所）には注意しましょう。

ターミナル（コマンドプロンプト）を扱ったり、プロクラミングをする上で、自分がどの位置にいてどのファイルを参照したいのか、どのファイルを実行したいのか、の「パス(path)」は正確に指定する必要があります。（相対パスと絶対パスについてを参照）

Node.jsは、サーバーサイドJavaScriptです。サーバーサイドとは大げさに聞こえるかもしれませんが、要するにマシン上（PC上）で直接実行可能なJavaScriptになります。皆さんがJavaScriptに対して持たれているイメージは、HTMLのDOM（tag）を操作する為のもの、HTMLページ上で出来ることを少し増やすためのスクリプト言語、などかもしれませんが（＝ブラウザ上で実行するもの）、そうではなく、PC上で直接実行可能にしたJavaScriptの実装がNode.jsになります。

Node.jsでwebアプリケーションとは

伝統的な方法でwebアプリケーションを構築するとしたら、ApacheとPHPの組み合わせを思い浮かべるかもしれません。

しかし、Node.jsを用いた場合は、httpリクエストを処理する機構が標準で備えられているため、Node.js単体で十分にwebアプリケーションを構築することが可能です。（もっと高度なことをする場合、ApacheやNginxを組み合わせることもあるでしょう）

Node.jsにwebサーバー構築の機能は備わっているものの、独力でそれを使いこなすためには時間と労力が必要になります。

そのため、webアプリケーションを便利に構築するためのフレームワークが多数作られています。ここでは、その内の一つのExpressを使ってwebアプリケーションを実現します。
